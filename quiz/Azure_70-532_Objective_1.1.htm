How is Traffic Manager implemented to route Web App traffic to an appropriate website endpoint?
It acts as a Proxy where all website traffic flows through, then gets routed to the correct endpoint.
It's configured at the DNS level, so that DNS resolution directs which endpoint to route traffic to.
Which methods does Traffic Manager support for implementing Load Balancing?
Automatic
Round Robin
Failover
Performance
What is Failover Load Balancing?
Once a request fails on an endpoint, it is then automatically redirected to a different endpoint.
Traffic is directed to Healthy endpoints as the result of monitoring, so that traffic is not directed to a server that is in a state of failure.
What is Round Robin Load Balancing?
Each traffic request is directed to the next endpoint in the list of endpoints so traffic is evenly distrubuted across all endpoints.
Traffic is randomly distributed to endpoints so traffic is evenly distributed across all endpoints.
How is Performance Load Balancing performed?
Traffic is routed to the endpoint with the VM that has the most available CPU/Memory available.
Traffic is routed via DNS to the "closest" endpoint based on the lowest latency between the client and the Azure Data Center hosting the endpoint.
Traffic is routed via DNS to the endpoint with the fastest response times.
